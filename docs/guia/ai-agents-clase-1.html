1) ¿Qué es un agente?

Un agente es un sistema que percibe, razona y actúa para lograr objetivos en un entorno.

  • Reactivo: responde a estímulos.
  • Deliberativo: planifica y evalúa.
  • Colaborativo / Multiagente: coordina con otros agentes.

Loop de un agente

Percepción → Razonamiento → Acción → Evaluación → (repite)

Ej.: revisar emails, extraer facturas, actualizar BBDD, notificar resultados.

2) LLM vs Asistentes vs Agentes

ConceptoQué haceEjemplos
LLMGenera y transforma textoLlama, Mistral, Gemma
AsistenteGestión de contexto + herramientas limitadasChat assistants
AgenteToma decisiones y ejecuta acciones hacia un objetivoCrewAI, AutoGen

Salto clave: del prompt al plan; del texto a la acción; del contexto estático a memoria y herramientas.

3) Herramientas (Tools)

Una herramienta es una interfaz (función, API o servicio) que el agente invoca para interactuar con datos o realizar acciones.

Web Search DB SQL Mail IMAP/Gmail OCR PDF/Imagen Storage S3/MinIO HTTP APIs

  • Open WebUI: interfaz moderna para chat/agents.
  • Ollama: motor de modelos locales (Llama, Mistral, Gemma...).
  • Tools: integraciones hacia “el mundo real”.

6) Open WebUI Tools vs MCP Tools

AspectoOpen WebUI ToolMCP Tool
ModeloFunción local (SDK WebUI)Servicio externo estandarizado (MCP)
EjecuciónDentro de WebUIVía servidor MCP (proxy)
InteroperabilidadEn el entorno WebUIEntre agentes y plataformas
SeguridadLocal, menos aislamientoAislamiento por proceso/servidor
ReusoOrientado a proyectoOrientado a ecosistema
MCP = Protocolo para conectar agentes con herramientas de forma portable, segura y reutilizable.

7) Extender Open WebUI

Pasos típicos:

  1. Definir la herramienta (tools/): Python/JS
  2. Describir el contrato (inputs/outputs)
  3. Registrar en la configuración (JSON)
  4. Probar en WebUI con un flujo guiado

Ejemplo (pseudo-Tool):

{
  "name": "ocr_image_tool",
  "description": "Extrae texto de imágenes",
  "schema": {"path": "string"},
  "handler": "tools.ocr_image:run"
}

Ideas de integración: Email, MinIO/S3, DB SQL, HTTP APIs, OCR (PDF/imagen), planificador.

Ejemplo conceptual · MCP server (mínimo)

# server.py (borrador mínimo conceptual)
from fastmcp import MCPServer, tool

app = MCPServer("demo-mcp")

@tool()
def email_count(inbox: str) -> int:
    "Devuelve el número de emails en un buzón"
    # ... lógica/IMAP ...
    return 42

if __name__ == "__main__":
    app.run()  # expone las tools vía MCP

Un cliente (p.ej. Open WebUI) podría descubrir y usar estas herramientas MCP.
